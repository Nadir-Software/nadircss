Buttons

Buttons let people take action and make choices with one tap. The default button should be used for important actions such as internal links or navigation.

Default button

Default text buttons should always have accompanying icons. Icons should be relevant to the action and convey the result of the action. Use Material Symbols Rounded for icons.

Default button with icon

Sometimes space constraints or stylistic choices require icon-only buttons. These should only be used in circumstances where the action is obvious to the user.

Default button without text

Some actions are less important or require less emphasis than others. For this, the button should be assigned a subtle style.

Subtle button

Use the subtle style when linking to external products or services. These are not associated with your product or service so should feel separate to avoid confusion. The accompanying icon should be the logo or emblem of the external product or service, preferably in white or dark gray.

A button's appearance should reflect its state, therefore there are different types of buttons for each state. Sometimes an action is unavailable or has failed to complete. In this case, the button should turn red.

Error button

There are two instances in which this button type should be used:

  • The action is not available
  • The action failed to complete after user interaction

If the action is not available, the button should be assigned this style and have an disabled attribute, meaning that the user cannot interact with the button.

If the action failed to complete, the button should be assigned this style and the disabled attribute for 1 second before resetting.

Another state is "Success". Buttons should have this style to signify that an action was successful. As such, the style should be assigned for 1 second before being reset to default.

Success button

If applicable, the text should also change to reflect the result of the action.

  • "Create new" changes to "Created new"
  • "Copy" changes to "Copied"
  • "Save" changes to "Saved"

Exclamation marks should not be used at the end of changed text. They convey informality, which does not conform with the design foundations of NadirCSS.

The final state is "Warning". Buttons should have this style to indicate that an action entails risk or uncertainty, or that it may have serious consequences. This style should be permanent.

Warning button

Cards

Cards display content and actions about a single subject. The default card should be used to display related and important information.

Subtle buttons, search, and text inputs have the same background color as subtle cards. This means that a subtle button, search, or text input on a subtle card will be difficult to distinguish. As such, a 1px border of rgb(100, 100, 100) should be added.

Navbar

Navbars display the title of the current produce or service, helping users to locate and navigate different products or services in an ecosystem. They are always located at the top of a page. This consistency aids in navigation.

Navigate to the top of the page to see a navbar.

Search

Search lets people enter a keyword or phrase to get relevant information. Search consists of a decorative icon and a text input field. Search also follows states, which correspond to the retrieval of information.

Search

Text inputs

Text inputs let users enter text into a UI. The default text input should be used for important information.

Default text input

Subtle text input should be used for less important information or to match the style of a card.

Subtle text input

Error text input should be used when the user has inputted invalid or incorrect information. This style should be applied for 1 second after the user's input was evaluated.

Error text input

Success text input should be used in two different ways for two different scenarios:

  • The user previously inputted invalid text and then inputted valid text
  • The user inputted valid text into a validity checker, for example in the Format Checker service
